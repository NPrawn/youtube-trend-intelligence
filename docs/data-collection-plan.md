# 데이터 수집 계획

## 1. 수집 목적

수집의 목적은 단순 저장이 아니라 트렌드 분석에 필요한 시계열 데이터를 확보하는 것이다.

즉, 특정 시점의 현재 상태만 보는 것이 아니라, 시간이 지나면서 영상 · 채널 · 키워드가 어떻게 변하는지를 추적할 수 있어야 한다.

## 2. 초기 데이터 소스

초기 데이터 소스는 YouTube Data API이다.

이 프로젝트의 목적은 트렌드 분석이며, 유튜브는 공개 API를 제공하기 때문에 초기 수집 대상으로 선택한다.

## 3. 사용할 API

### videos.list
용도:
- 지역 / 카테고리별 인기 영상 수집
- 영상 통계 및 메타데이터 조회

### channels.list
용도:
- 채널 메타데이터 및 통계 조회

### search.list
용도:
- 특정 키워드 관련 최근 영상 수집
- 추출한 키워드 검증 및 확장

초기 MVP에서는 videos.list와 channels.list 중심으로 시작한다.
`search.list`는 quota 비용이 높기 때문에 후순위로 둔다.

## 4. 수집 대상

### 4.1 영상
- 지역 / 카테고리별 인기 영상
- 특정 검색 결과 상위 영상
- 향후 키워드 관련 영상

### 4.2 채널
- 인기 영상에 포함된 채널
- 최근 반복 등장하는 채널
- 검색 결과 기반 채널

### 4.3 키워드
- 영상 제목
- 영상 설명
- 태그
- 추후 검색 키워드 후보

## 5. 수집 전략

### 5.1 초기 전략
초기에는 `videos.list(mostPopular)` 중심 수집을 먼저 구현한다.

이유:
- 구현 난도가 낮다.
- 핵심 기능 검증이 가능하다.
- quota 관리가 상대적으로 쉽다.

초기 표본의 기본 단위는 다음과 같다.

- `regionCode`
- `videoCategoryId`
- `collectedAt`
- API 응답 내 순위

### 5.2 확장 전략
이후 영상 제목/설명/태그에서 키워드를 추출하고, 이를 바탕으로 search.list를 제한적으로 활용한다.

확장 시에도 다음 원칙을 지킨다.

- 검색 키워드는 내부 점수 상위 후보만 사용한다.
- region / language 범위를 고정한다.
- 검색 호출 수는 일일 quota 예산 안에서 제한한다.
- 검색 결과는 "탐색 확장 표본"으로 별도 출처를 기록한다.

## 6. 수집 주기 초안

### 인기 영상
- 10분 또는 30분 단위

### 채널 정보
- 30분 또는 1시간 단위

### 키워드 검색 수집
- 1시간 또는 3시간 단위

초기에는 과도한 수집보다 안정적으로 반복 수집이 가능한 구조를 우선한다.
MVP에서는 quota 안정성을 위해 수집 주기를 보수적으로 잡는다.

## 7. 저장 전략

### 원본 스냅샷
각 수집 시점의 원본 응답을 기반으로 필요한 필드를 정규화하여 저장한다.

목적:
- 재집계 가능
- 디버깅 가능
- 과거 상태 비교 가능

### 집계 데이터
원본 스냅샷을 기반으로 분 / 시간 / 일 / 주 / 월 / 년 단위 집계를 생성한다.

### 출처 메타데이터
각 수집 결과에는 최소한 다음 정보를 함께 저장한다.

- source_endpoint
- source_region
- source_category
- source_query
- source_rank
- source_type

## 8. 키워드 후보 발굴 전략

### 1단계
영상 제목에서 후보 단어 추출

### 2단계
설명 / 태그까지 확장

### 3단계
불용어 제거, 중복 표현 정규화

### 4단계
등장 빈도와 관련 영상 점수를 기준으로 후보 키워드 선정

초기에는 단순한 방식으로 시작하고, 이후 점진적으로 고도화한다.
키워드는 공식 검색어가 아니라 수집된 메타데이터에서 계산한 내부 지표임을 명시한다.

## 9. 숏폼 / 롱폼 분류 방식

초기에는 영상 길이를 기준으로 한 휴리스틱 분류를 사용한다.

- short-form: 3분 이하
- long-form: 3분 초과

영상 메타데이터에서 duration을 파싱하여 저장 시점에 분류한다.
이 값은 공식 Shorts 식별값이 아니므로, 분석 화면에서는 "length-based short"로 해석할 수 있어야 한다.

## 10. 채널 후보 선정 방식

채널 후보는 다음 기준으로 수집한다.

- 인기 영상에 포함된 채널
- 최근 반복적으로 등장하는 채널
- 향후 검색 결과에 자주 등장하는 채널

이 방식은 관측된 채널을 중심으로 확장하는 전략이며, 유튜브 전체 신규 채널을 전수 탐지하는 방식은 아니다.

## 11. quota 대응 전략

- 초기에는 videos.list 중심으로 간다.
- search.list는 제한된 키워드 후보에만 사용한다.
- 무작위 전수 탐색은 하지 않는다.
- 수집 범위를 명확하게 제한한다.
- 일일 quota 예산을 먼저 정하고 수집 스케줄을 맞춘다.
- 인기 영상 수집과 검색 수집의 quota를 분리 관리한다.

## 12. 수집 시 유의사항

- API 키는 환경변수로 관리한다.
- 수집 실패 시 재시도 정책을 둔다.
- 동일 시점 중복 저장을 방지한다.
- collectedAt 기준을 일관되게 관리한다.
- API 응답 출처와 수집 범위를 함께 저장한다.
