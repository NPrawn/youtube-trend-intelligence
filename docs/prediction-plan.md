# 예측 기능 계획

## 1. 목적

예측 기능의 목적은 미래를 완벽히 맞히는 것이 아니라, 현재 상승 중인 대상들 중에서 앞으로 더 커질 가능성이 높은 대상을 우선순위화하는 것이다.

즉, 사용자가 지금 뜨는 것뿐 아니라 곧 더 뜰 가능성이 있는 것까지 빠르게 파악할 수 있도록 돕는 것이 목표다.

## 2. 예측 대상

초기 예측 대상은 다음 세 가지로 정의한다.

- video
- channel
- keyword

## 3. 예측 창

초기 예측 창은 다음과 같이 정의한다.

- 2시간
- 6시간
- 24시간

## 4. 예측 질문

예측은 아래 질문 형태로 생각한다.

- 이 영상은 앞으로 2시간 내 더 상승할 가능성이 있는가
- 이 채널은 오늘 안에 더 큰 상승을 보일 가능성이 있는가
- 이 키워드는 앞으로 6시간 동안 계속 확산될 가능성이 있는가

단, 이 질문은 모두 수집 표본 내부에서의 상대적 상승 가능성을 뜻한다.

## 5. 영상 예측 피처 초안

- 최근 hot_score
- 최근 rising_score
- 최근 조회수 증가율
- 최근 좋아요 증가율
- 최근 댓글 증가율
- 최근 순위 변화
- 최근성
- short / long 여부
- source 유형

## 6. 채널 예측 피처 초안

- 최근 영상 평균 점수
- 최근 영상 상승 비율
- 최근 인기 영상 포함 개수
- 최근 구독자 증가량
- 최근 조회수 증가량
- 신생 채널 여부
- short-heavy / long-heavy 여부
- 표본 내 반복 등장 횟수

## 7. 키워드 예측 피처 초안

- 최근 등장 빈도
- 최근 관련 영상 수
- 최근 관련 채널 수
- 현재 hot_score
- 현재 rising_score
- 직전 구간 대비 변화량
- 최근성
- 출처 다양성

## 8. 초기 구현 방식

초기에는 복잡한 모델보다 해석 가능한 방식으로 시작한다.

예:
- 규칙 기반 점수
- logistic regression
- gradient boosting

첫 단계에서는 “정확도 극대화”보다 “동작하는 예측 파이프라인 구축”을 우선한다.
라벨 역시 단순하게 정의한다.

예:
- 2시간 후 같은 form / source 범위 안에서 rank 상승
- 6시간 후 hot_score 상위 구간 진입
- 24시간 후 관측 유지 여부

## 9. 예측 결과 저장

예측 결과는 prediction_result 테이블에 저장한다.

저장 목적:
- API 제공
- 모델 성능 비교
- 추후 재학습 또는 재평가

## 10. 평가 방식

예측 성능은 다음 관점에서 평가한다.

- 실제 상승 여부와의 일치
- top N hit rate
- precision / recall
- 예측 창별 성능 비교
- source / form 별 성능 비교

## 11. 확장 방향

향후에는 다음 방향으로 확장할 수 있다.

- 시계열 모델 적용
- entity 간 연관성 반영
- short-form / long-form별 별도 모델
- 키워드-영상-채널 관계 그래프 반영

## 12. 주의점

- 초기 예측은 단순하고 설명 가능해야 한다.
- 예측 결과는 “확률적 추정”으로 해석해야 한다.
- 예측 기능이 핵심 서비스 가치를 보조하는 형태여야 하며, 전체 프로젝트의 발목을 잡아서는 안 된다.
- 표본 범위를 벗어난 대상에 대한 예측으로 오해되지 않도록 해야 한다.
