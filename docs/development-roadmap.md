# 개발 로드맵

## 1. 개발 원칙

이 프로젝트는 한 번에 완성형을 만들기보다, 항상 실행 가능한 상태를 유지하면서 단계적으로 확장한다.

개발 순서는 다음 원칙을 따른다.

- 문서화 먼저
- 작은 검증 먼저
- 수집부터 구현
- 저장과 조회를 먼저 안정화
- 이후 분석과 예측 확장

## 2. 1단계: 기획 및 API 검증

### 목표
- 프로젝트 문서 작성
- YouTube API 응답 구조 확인
- 수집 가능 데이터 확인

### 해야 할 일
- README 작성
- 핵심 docs 문서 작성
- videos.list 테스트
- channels.list 테스트
- duration 파싱 테스트

### 완료 조건
- 핵심 문서가 작성되어 있다.
- 인기 영상 API 응답을 확인했다.
- 숏폼 / 롱폼 분류 가능성을 확인했다.

## 3. 2단계: 인기 영상 수집 및 저장

### 목표
- 인기 영상 수집기 구현
- 영상 / 영상 스냅샷 저장

### 해야 할 일
- 수집 스크립트 또는 수집 서비스 구현
- video 테이블 생성
- video_snapshot 테이블 생성
- 수집 주기 설정

### 완료 조건
- 인기 영상 데이터를 주기적으로 저장할 수 있다.
- 동일 영상의 중복 저장 정책이 정리되어 있다.
- 스냅샷 데이터가 누적 저장된다.

## 4. 3단계: 조회 API 구현

### 목표
- 최근 Hot Video 조회 API 제공
- 기간 필터 제공
- short / long 필터 제공

### 해야 할 일
- trend 조회 로직 구현
- /trends/videos 구현
- 상세 조회 API 구현

### 완료 조건
- 최근 Hot Video 조회가 가능하다.
- short / long 필터가 동작한다.
- 기간별 조회가 가능하다.

## 5. 4단계: 채널 분석 기능 구현

### 목표
- 채널 후보 수집
- 채널 스냅샷 저장
- Rising Channel 계산

### 해야 할 일
- channel / channel_snapshot 설계 및 구현
- 인기 영상 기반 채널 후보 추출
- 최근 성과 기반 채널 점수 계산
- /trends/channels 구현

### 완료 조건
- Rising Channel 조회가 가능하다.
- 채널 상세 조회가 가능하다.
- 기본 성장 지표가 동작한다.

## 6. 5단계: 키워드 추출 기능 구현

### 목표
- 영상 제목 / 설명 기반 키워드 추출
- Hot Keyword / Rising Keyword 계산

### 해야 할 일
- 키워드 추출 로직 구현
- 불용어 제거
- keyword_snapshot 생성
- /trends/keywords 구현

### 완료 조건
- 최근 자주 등장하는 키워드를 보여줄 수 있다.
- Rising Keyword를 보여줄 수 있다.

## 7. 6단계: 리바이벌 및 고도화

### 목표
- Revival Video 탐지
- Breakout New Channel 계산
- 채널 분석 고도화

### 해야 할 일
- 과거 대비 최근 반응 증가 기준 설계
- 신생 채널 판단 기준 설계
- Revival Video API 확장

### 완료 조건
- 리바이벌 대상 식별이 가능하다.
- 신생 채널 흐름을 보여줄 수 있다.

## 8. 7단계: 예측 기능 구현

### 목표
- 단기 예측 피처 생성
- 예측 점수 계산
- prediction API 제공

### 해야 할 일
- 피처 정의
- baseline 모델 구현
- 예측 결과 저장
- /predictions 구현

### 완료 조건
- 앞으로 더 뜰 가능성이 있는 대상을 보여줄 수 있다.
- 최소 1개 예측 시나리오가 동작한다.

## 9. 8단계: 운영 및 성능 개선

### 목표
- Redis 캐시 도입
- 배치 안정화
- 성능 개선
- 모니터링 추가

### 해야 할 일
- 인기 조회 결과 캐싱
- 집계 배치 정리
- 장애 처리 정책 정리
- 로그 및 메트릭 정리

### 완료 조건
- 조회 성능이 개선된다.
- 반복 조회에 대한 부담이 줄어든다.
- 장애 상황을 추적할 수 있다.
