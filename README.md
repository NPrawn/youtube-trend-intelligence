# youtube-trend-intelligence

빠르게 변화하는 트렌드를 쉽고 직관적으로 파악할 수 있도록 돕는 시계열 기반 트렌드 분석 서비스

## 프로젝트 한 줄 소개

youtube-trend-intelligence는 공개 데이터를 활용해 영상 / 채널 / 키워드의 변화 흐름을 분석하고, 현재 뜨는 흐름과 새롭게 형성되는 흐름을 한눈에 보여주는 서비스이다.

## 프로젝트 배경

온라인 트렌드는 빠르게 바뀐다.
특정 주제는 짧은 시간 안에 급부상하기도 하고, 이미 지나간 콘텐츠가 다시 주목받기도 한다.
하지만 이러한 변화를 직접 파악하려면 여러 콘텐츠를 지속적으로 탐색해야 하며, 이는 많은 시간과 피로도를 요구한다.

특히 마케팅팀, 콘텐츠 기획자, 인플루언서, 미디어 운영자처럼 트렌드 파악이 실제 업무나 성과와 직접 연결되는 사용자에게는 단순한 인기 콘텐츠 목록만으로는 부족하다.
이들이 필요로 하는 것은 현재 많이 보이는 결과 자체보다, 어떤 흐름이 새롭게 만들어지고 있는지, 어떤 채널이 성장하고 있는지, 어떤 키워드가 반복적으로 확산되는지에 대한 정보다.

이 프로젝트는 이러한 문제를 해결하기 위해 시작되었다.

## 프로젝트 목적
이 프로젝트의 핵심 목적은 유튜브 자체를 분석하는 것이 아니라, 빠르게 변화하는 트렌드를 쉽게 파악할 수 있도록 돕는 것이다.

현재는 공개 API를 제공하는 유튜브를 초기 데이터 소스로 사용하여 다음과 같은 정보를 제공하는 것을 목표로 한다.

- 지금 주목받는 영상
- 최근 빠르게 떠오르는 채널
- 반복적으로 등장하는 키워드
- 과거 콘텐츠의 리바이벌 현상
- 숏폼과 롱폼의 흐름 차이
- 앞으로 더 확산될 가능성이 있는 대상

## 현실적인 범위

이 프로젝트는 유튜브 전체를 완전하게 감시하는 시스템이 아니다.
초기 버전은 YouTube Data API로 수집 가능한 공개 데이터만 사용하며, 다음 범위 안에서 동작한다.

- `videos.list(mostPopular)` 기반 인기 영상 표본 수집
- 수집된 영상과 연관 채널의 시계열 스냅샷 분석
- 제한된 `search.list` 사용을 통한 키워드 확장 수집
- 내부 표본 집합 안에서의 Hot / Rising / Keyword / Channel 분석

따라서 이 서비스의 결과는 "유튜브 전체의 공식 실시간 트렌드"가 아니라, "수집 표본 안에서 계산한 트렌드 인텔리전스"로 해석해야 한다.

## 주요 타겟 사용자

- 마케팅팀 / 브랜드 마케터
- 콘텐츠 기획자 / 미디어 운영자
- 인플루언서 / 크리에이터
- 트렌드 분석이 필요한 실무자
- 현재 흐름을 파악하고 싶은 일반 사용자

주요 타겟은 일반 대중 전체가 아니라, 트렌드 파악이 실제 의사결정과 연결되는 사용자층이다.

## 핵심 기능

- Hot Video: 현재 강하게 주목받는 영상
- Rising Video: 최근 빠르게 상승 중인 영상
- Revival Video: 과거 영상 중 다시 떠오르는 영상
- Rising Channel: 최근 빠르게 성장하는 채널
- Breakout New Channel: 신생이지만 빠르게 반응을 얻는 채널
- Hot Keyword: 반복적으로 등장하는 트렌드 키워드
- Short-form / Long-form Trend: 숏폼과 롱폼 흐름 분리
- Short-term Prediction: 단기 상승 가능성 예측

## 서비스가 제공하는 가치

- 여러 콘텐츠를 직접 탐색하지 않아도 트렌드를 빠르게 파악할 수 있다.
- 단순 인기 목록이 아니라 변화 흐름을 볼 수 있다.
- 영상 단위뿐 아니라 채널과 키워드 단위로 흐름을 이해할 수 있다.
- 마케팅, 콘텐츠 기획, 채널 운영에 활용할 수 있다.
- 데이터 기반으로 트렌드를 해석하는 경험을 제공한다.

## 차별점

- 단순 인기 콘텐츠 나열이 아니라 변화량 기반 분석
- 영상뿐 아니라 채널 성장 흐름 분석
- 수집한 데이터로부터 키워드를 직접 추출
- 숏폼 / 롱폼 분리 분석
- 과거 콘텐츠 리바이벌 탐지
- 단기 확산 가능성 예측

## 데이터 한계

- Shorts 여부는 초기에는 공식 식별값이 아니라 길이 기반 휴리스틱으로 분류한다.
- Shorts와 일반 영상은 조회수 집계 방식 차이가 있을 수 있으므로 직접 비교 점수는 보정이 필요하다.
- 키워드는 유튜브의 공식 검색어 순위가 아니라 수집된 영상 메타데이터에서 계산한 내부 지표다.
- Rising / Revival / Breakout은 수집된 후보군 내부에서만 계산할 수 있다.
- 채널 구독자 증감은 공개 스냅샷 기반 추정이며, 숨김 구독자 수 또는 반올림 정책의 영향을 받을 수 있다.

## 초기 데이터 소스

이 프로젝트의 본질은 트렌드 분석 서비스이며, 유튜브는 이를 구현하기 위한 초기 데이터 소스이다.
유튜브를 선택한 이유는 공개 API를 제공하며, 영상 / 채널 / 키워드 단위의 변화 흐름을 수집하고 분석하기에 적합하기 때문이다.
다만 API quota와 검색 범위 제약이 있으므로, 초기에는 전수 감시가 아니라 표본 기반 수집 전략으로 시작한다.

## 기술 스택

- Backend: Spring Boot
- Database: PostgreSQL or MySQL
- Cache: Redis
- Batch: Spring Batch
- External API: YouTube Data API
- Infra: Docker
- Optional: Python (키워드 추출 및 예측 실험)

## MVP 범위

1차 목표는 아래 기능까지 구현하는 것이다.

- YouTube Data API 연동
- 인기 영상 수집
- 영상 메타데이터 저장
- 숏폼 / 롱폼 분류
- 최근 인기 영상 조회 API
- 기본 Rising Channel 분석

## 개발 단계
1. 서비스 기획 및 지표 정의
2. API 검증 및 수집 구조 확인

4. 데이터 저장 및 조회 API 구현
5. 채널 분석 기능 구현
6. 키워드 추출 기능 구현
7. 예측 기능 확장
